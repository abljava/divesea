# План реализации проекта NFT Marketplace

## Анализ задания

### Описание проекта
Разработка адаптивной одностраничной веб-страницы NFT маркетплейса с:
- Hero секцией с призывом к действию и блоком статистики (объединены в один компонент)
- Слайдером "Weekly - Top NFT" с данными из API
- Нижним блоком с призывом к действию (CTA секция)
- Футером с навигацией и социальными ссылками

### Технические требования
- **Фреймворк**: React или Next.js
- **State Management**: Redux или Redux Toolkit
- **Стилизация**: SCSS
- **Анимации**: GSAP или Framer Motion
- **API**: https://api.coingecko.com/api/v3/nfts/list
- **Адаптивность**: Desktop / Tablet / Mobile

### Решение: React vs Next.js

**Выбранное решение: React (с Vite)**

**Обоснование:**
1. **Тип проекта**: Одностраничное приложение (SPA) без требований к SSR/SSG
2. **SEO**: Не критично для данного типа проекта (NFT маркетплейс обычно требует авторизации)
3. **Простота разработки**: React + Vite быстрее в настройке и разработке
4. **Производительность**: Vite обеспечивает быструю разработку и оптимизированную сборку
5. **Деплой**: Проще деплой статического SPA (Netlify, Vercel, GitHub Pages)
6. **Размер бандла**: Меньше overhead для простого landing page

**Когда выбрать Next.js:**
- Если нужен SEO для публичных страниц
- Если планируется расширение до многостраничного приложения
- Если требуются SSR/SSG функции

## Поэтапный план реализации

### Этап 1: Инициализация проекта (1-2 часа)

1. **Создать React проект с Vite**
   ```bash
   npm create vite@latest . -- --template react-ts
   ```

2. **Установить зависимости**
   - React, React DOM
   - Redux Toolkit (`@reduxjs/toolkit`)
   - React Redux (`react-redux`)
   - SCSS (`sass`)
   - GSAP (`gsap`)
   - React Icons (`react-icons`) для иконок

3. **Настроить TypeScript**
   - Проверить `tsconfig.json`
   - Настроить пути (paths) если необходимо

4. **Настроить структуру папок**
   - Создать все необходимые директории
   - Создать базовые файлы

### Этап 2: Настройка Redux Store (2-3 часа)

1. **Настроить Redux Toolkit store**
   - Создать `store/index.ts`
   - Настроить middleware (thunk)

2. **Создать NFT slice**
   - State: список NFT, статус загрузки, ошибки
   - Actions: fetchNFTs, setLoading, setError
   - Reducers для обработки состояний

3. **Создать API service**
   - Функция для запроса к CoinGecko API (использовать `fetch`)
   - Обработка ошибок
   - Типизация ответа API

4. **Интегрировать API с Redux**
   - Создать async thunk для загрузки NFT
   - Обработать случаи успеха/ошибки

### Этап 3: Утилиты и типы (1-2 часа)

1. **Создать TypeScript типы**
   - NFT interface (на основе API)
   - Состояния загрузки
   - Пропсы компонентов

2. **Создать утилиты**
   - `generateTimer()` - генерация случайного времени до окончания (07h 09m 12s формат) для каждой карточки
   - `generateBid()` - генерация случайной ставки для каждой карточки (формат с иконкой алмаза)
   - `getRandomImage()` - выбор случайного изображения из набора

3. **Создать хук для таймера**
   - `useCountdownTimer()` - хук для реализации работающего таймера отсчета с обновлением каждую секунду

4. **Подготовить изображения**
   - Создать набор изображений для NFT карточек (можно использовать placeholder или градиенты)
   - Сохранить в `public/images/`

### Этап 4: Базовые компоненты (3-4 часа)

1. **Header компонент**
   - Логотип (wavy line icon)
   - Навигационное меню (DISCOVER, CREATORS, SELL, STATS)
   - Кнопка "CONNECT WALLET"
   - Адаптивное меню для мобильных устройств
   - **Закрепленный хедер (fixed position)** - хедер остается видимым при скролле
   - Фон хедера задается через CSS (без динамического изменения при скролле)
   - Реализация: `position: fixed` в SCSS стилях

2. **Button компонент (общий)**
   - Варианты стилей (primary, secondary, outline)
   - Поддержка разных размеров
   - Анимация при наведении

3. **Timer компонент**
   - Отображение времени в формате "07h 09m 12s"
   - Обновление каждую секунду (опционально)
   - Стилизация

### Этап 5: Hero секция с блоком статистики (3-4 часа)

1. **Hero компонент (объединенный с блоком статистики)**
   - **Левая часть (текстовый блок):**
     - Заголовок "Discover And Create NFTs" (h1)
     - Описание с выделенным "$20 bonus"
     - Две кнопки: "EXPLORE MORE" и "CREATE NFT"
     - Блок статистики (три блока):
       - 430K+ Art Works
       - 159K+ Creators
       - 87K+ Collections
   
   - **Правая часть (визуальные элементы):**
     - Два изображения (градиентные артворки):
       - Основное изображение (stacked ovals 3D artwork)
       - Второе изображение (blurred gradient square)
     - Стрелка между изображениями (hand-drawn style arrow)
     - Точки/паттерн на фоне (dotted grid pattern)

2. **Анимация появления (GSAP):**
   - **Текстовый блок (с задержкой 200-300мс между элементами):**
     - Сначала появляется h1 заголовок (fade in + slide up)
     - Затем текст описания (fade in)
     - Затем кнопки (fade in)
     - Затем блок со статистикой (fade in + счетчики с анимацией)
   
   - **Визуальные элементы:**
     - С задержкой 200-300мс две картинки появляются из-за правой границы экрана (slide in from right)
     - Картинки встают на свое место
     - После плавно появляется стрелка (opacity 0% → 100%)

3. **Стилизация Hero**
   - Адаптивная вёрстка (desktop/tablet/mobile)
   - Flexbox/Grid для расположения элементов
   - Градиенты для изображений (или использование SVG/изображений)
   - Адаптивная сетка для статистики

### Этап 6: NFT Slider (4-5 часов)

1. **NFTCard компонент**
   - Изображение NFT (случайное из набора)
   - Таймер отсчета (countdown) в правом верхнем углу карточки (формат: "07h 09m 12s")
     - Реализовать работающий таймер с отсчетом времени
     - Каждая карточка имеет свое случайное время окончания
   - Название NFT (из API - поле `name`)
   - Current bid с иконкой алмаза (рандомизированное значение для каждой карточки)
   - Кнопка "PLACE BID"
   - Hover эффекты

2. **NFTSlider компонент**
   - Интеграция с Redux для получения данных
   - Отображение карточек в горизонтальном слайдере
   - **Бесконечная прокрутка (обязательно)** - слайдер должен зацикливаться
   - Навигационные стрелки (влево/вправо) для переключения слайдера
   - **Поддержка свайпа и перетаскивания мышью:**
     - Touch events для мобильных устройств (swipe)
     - Mouse drag (перетаскивание мышью) для десктопа
     - Обработка начала, перемещения и окончания перетаскивания
   - Адаптивное количество видимых карточек:
     - Desktop: 3-4 карточки
     - Tablet: 2 карточки
     - Mobile: 1 карточка
   - Карусель с плавной анимацией (GSAP или CSS transitions)

3. **Кастомный хук useNFTSlider**
   - Логика навигации слайдера
   - Реализация бесконечной прокрутки
   - Обработка touch событий (swipe на мобильных)
   - Обработка mouse events (перетаскивание мышью на десктопе)
   - Расчет позиций и индексов для бесконечной прокрутки
   - Обработка начала, перемещения и окончания перетаскивания

## Особенности реализации

### API Integration
- Использовать встроенный `fetch` API для запросов к CoinGecko API
- Обработать CORS (если потребуется прокси)
- Обработать rate limiting API
- Использовать только поле `name` из ответа API

### Слайдер
- Реализовать кастомный слайдер (не использовать готовые библиотеки, если не требуется)
- **Бесконечная прокрутка (обязательно)** - слайдер должен зацикливаться
- Поддержка touch/swipe для мобильных устройств
- Поддержка перетаскивания мышью (mouse drag) для десктопа
- Плавная анимация перехода
- Навигационные стрелки (переключение по кнопкам)
- Адаптивное количество видимых карточек

### Генерация данных
- Таймер: случайное время от 1 до 24 часов
- Ставка: случайное значение от 0.1 до 10 ETH (или использовать формат с алмазом)
- Изображения: массив путей к изображениям, случайный выбор

### Адаптивность
- Mobile-first подход или Desktop-first (на усмотрение)
- Использовать SCSS mixins для media queries
- Тестирование на реальных устройствах
- Оптимизация для touch интерфейсов
